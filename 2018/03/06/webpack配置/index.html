webpack 的使用

一：初级使用：webpack 需要打包的文件名 打包后的文件名(js文件)

​ 导出一个模块 module.exports = 需要导出的对象

导入一个模块 `var 模块名 = require(文件路径)`

​ webpack 打包命令 webpack 需要打包的文件名 输出的文件名

​ 例如: webpack main.js build.js

二：配置webpack.config.js文件.

1.默认配置文件名称webpack.config.js

2.使用webpack打包

  • 命令行直接输入 “webpack”
  • 如果是其他改过名字的配置文件 则使用 ‘webpack –config 配置文件名’

三：各种loader的配置

1.初始化package.json

​ npm init

2.css-loader 是进行css解析，能够使浏览器正常识别js文件中导入的 css 文件

  • 安装css-loader以及style-loader

    npm install style-loader css-loader --save-dev

  • 在webpack.config.js文件中进行配置

  • 在main.js中导入css文件

require(‘文件路径’)

3.sass-loader是进行scss文件的解析

  • 安装sass-loader

npm install node-sass sass-loader style-loader css-loader –save-dev

  • 在webpack.config.js文件中进行配置
  • 在main.js中导入sass文件

require(‘文件路径’)

4.less-loader是进行less文件的解析

  • 安装less-loader

npm install less less-loader style-loader css-loader –save-dev

  • 在webpack.config.js文件中进行配置
  • 在main.js中导入less文件

require(‘文件路径’)

5.url-loader是进行url资源的解析

  • 安装url-loader

npm install url-loader file-loader –save-dev

  • 在webpack.config.js文件中进行配置
  • 在main.js中导入less文件

require(‘文件路径’)

6.webpack-dev-server的使用

  • 安装webpack-dev-server

npm install webpack webpack-dev-server –save-dev(不接版本号，默认安装最新版本)

  • 安装自动生成HTML文件的插件

npm install html-webpack-plugin –save-dev

  • 修改package.json文件
  • 在webpack.config.js文件中进行配置

  1. 引入html-webpack-plugin

    var htmlWP = require(‘html-webpack-plugin’)；

  2. 在modul.exports 中加入

  3. 在模板HTML文件中删除引入build.js的script脚本

  4. 运行 npm run dev

7.babel-loader的使用

babel-loader 是用来将es6语法转换成es5语法

  • 安装babel-loader

npm install babel-core babel-loader babel-preset-es2015 babel-plugin-transform-runtime –save-dev
